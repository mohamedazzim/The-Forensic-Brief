## What This Pattern Shows

This pattern captures a position effect in long code reviews: reviewers perform much better near the top of a diff than after line 600.

## Evidence

The technical design document's example describes five PRs in which reviewers caught 78% of issues in the first 200 lines and 0% past line 600.

## Test Guidance

Use long-diff review tests and compare detection rates before and after line 600. Keep the metric explicit in any regression check or review checklist.

# Mode: Drama (🎭)

USE WHEN: A real decision needs stress-testing.

RULE: Panel members argue the decision, not the user.

RENDER AS: Writers' room debate. Each round opens with "🎭 DRAMA — Round {n}: {stakes}". Panel members speak in first person with their name as prefix (e.g., "**Kelsey:** *We're trading latency for...*"). Characters disagree with each other BY NAME. The reader should hear distinct voices arguing, not a committee report. Tone is sharp, opinionated, professional — not polite consensus.

## Panel Composition

- Start with the triad (3 domain experts from Triad Contract).
- Add **The Anarchist** — challenges the premise. "Why are we solving this problem at all?"
- Add **The Saboteur** — finds the 3am failure scenario. "Picture it — Friday 5pm..."
- Add **CFO / Legal / Customer Voice** only when the decision affects money, compliance, or adoption.

All panel members get specific hands-on backgrounds for THIS problem.
Names come from the name pool in Triad Contract.

Ask: "Rename anyone, add a role, or shall we start?" Wait.

## Debate Levels (3 rounds)

**Level 1 — Positions:** Each panelist states what they believe. Sharp, opinionated.
**Level 2 — Challenges:** Attack weakest assumptions. Direct disagreements by name.
**Level 3 — Synthesis:** What survived scrutiny. Convergence toward decision.

STOP after each level. Ask before continuing.

## Round Format

```
🎭 DRAMA — Round {n}: {stakes}

**{Name}:** *{position in first person — sharp, opinionated}*
  → {specific point}
  → Challenge to {other Name}: {direct disagreement}

**{Name} (Anarchist):** *{premise challenge}*
  → Why we're solving the wrong problem
  → What we should be debating instead

**{Name} (Saboteur):** *{3am failure}*
  → "Picture it — Friday 5pm..."
  → Why the elegant solution breaks here

— Round {n} complete. Continue? Or steer?
```

## Convergence

When signal is strong enough across the panel:
- Produce a decision statement
- List alternatives rejected with reasons
- Action plan with owners and dependencies
- Remaining risks

## Deliverable
- Decision
- Rationale
- Alternatives rejected
- Action plan
- Risks
- Owners
- Dependencies

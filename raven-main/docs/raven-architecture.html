02 Flows
Three Things That Happen — Zero Manual Steps
Every operation is triggered by a git commit or a single Claude command.
FLOW A
Engine Update
When you improve raven-core
  • Edit anything in raven-core
  • git commit
  • pre-commit hook runs CVE + security checks
  • post-commit hook fires bundle.sh
  • Scripts copied to RAVEN
  • Scripts copied to raven-action
  • Andie skill synced
  • tools-landscape synced
  • Version stamp written everywhere
  • You push raven-core + RAVEN
FLOW B
New Project Setup
First time on a project
  • Run raven-setup.sh in project dir
  • Setup asks 8 questions
  • manifest.json created
  • Scripts + MCP + skills installed
  • Git hooks wired up
  • Auto-registers in ~/.raven/registry.json
  • Registry pushed to giggsoinc/raven-registry
  • Project now tracked — gets all future updates
FLOW C
Project Sync
After a VERSION bump
  • Run /raven-registry-sync --all
  • Reads ~/.raven/registry.json
  • Diffs each project's raven_version vs VERSION
  • Copies scripts → .claude/scripts/
  • Copies MCP → .claude/mcp/
  • Copies skills → .claude/skills/
  • Writes .raven/raven_version
  • git commit + push per project
  • Registry updated + pushed to GitHub

04 Flow C in Detail
Project Sync — Keeping All Registered Projects Current
Claude runs this via /raven-registry-sync. Works across any number of projects. Projects with a GitHub remote are pushed automatically. Local-only projects are committed but not pushed.

05 Registry
What the Registry Tracks — Current State
Lives at ~/.raven/registry.json locally. Backed up to giggsoinc/raven-registry on GitHub. Survives deleting any project repo. Survives wiping your machine (restore from GitHub clone).
